Introduction


Code-e-thon is designed in the way so that all the aspects of coding are covered. It contains bug fixing, error detection, problem solving (competitive coding) and miscellaneous decoding. Every thing is designed according to the participants point of view. We have found Hacker Rank UI nice for beginner so we will be using it for the event.

Flow of Event


1. Before the main event a briefing will be provided to the students about the event, so that participants get to know well about the event.
2. After the briefing the registration form will be circulated.
3. Event will start with the 1st round of the event.
4. Participants will be provided the system and the round will start. This round will run for 1 hour.
5. After this round break of 15 – 20 min will be provided.
6. Round 2 will start after break, with top 30% participants who qualified for round 2.
7. Round 2 will be of 2 hours.
8. After round 2 event will end and result will be declared later.

Who can Participate?


This event is open to 1st and 2nd year students of any recognized institution or college.

Rules and Regulations


1. Copying someone else’s code will lead to immediate disqualification from the whole event.
2. Discussing any problems (or any aspect of the problem) on any other platform on internet, if identified, could lead to disqualification of the participant from the contest.
3. Discipline should be maintained throughout the event.
4. Participants must coordinate properly with the event coordinators.
5. Asking anything related to the competition problems is not allowed during the contest.

Rounds


There will be 2 rounds:

Round 1:

Questions will be of level easy to medium level, this round will have four sets – A, B, C, D to avoid cheating.
Bug fixing: A compiling code will be provided but the output will be different. Participants will have to make necessary changes in code to get desired output. Participants need to write their change in comments in code.
Error detection: A code with some errors will be provided, Participants need to correct the code, if it complies then their question is done. All the changes made in code need to be explained in comments.
Competitive coding: Problems will be provided to the participants and they have to solve the problem by writing the correct code for the problem and submit it.
Miscellaneous decoding: problem related to ASCII code decoding, Bitwise Operators etc will be provided and participants will need to solve to score.

Round 2:

Questions will be of level medium to hard , this round will have 2 sets – J, K to avoid cheating. Rest will be same.

Marking scheme :

Round 1:
Error detection : 25 points
Bug fixing : 25 points
Competitive coding : 30 points
Miscellaneous coding : 20 points

Round 2:
Error detection : 50 points
Bug fixing : 50 points
Competitive coding : 60 points
Miscellaneous coding : 40 points
